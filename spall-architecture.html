spall Architecture

Three-crate OpenAPI CLI workspace

spall-cli

  • • Two-phase argument parsing
  • • Phase 1: registry scan (disable_help_flag)
  • • Phase 2: spec load, IR build, re-parse
  • • miette error reporting, exit codes 0-5

spall-config

  • • ApiRegistry discovery & config parsing
  • • Credential stack with SecretString
  • • Profiles & environment merging
  • • Never exposes raw credential strings

spall-core

  • • openapiv3 deserialization + manual $ref resolution
  • • Intermediate Representation (IR) for caching
  • • Dynamic clap Command builder from spec
  • • Lenient parsing, cycle detection, ordered maps
